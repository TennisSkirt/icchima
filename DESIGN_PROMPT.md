# 클로드에 붙여넣을 디자인 프롬프트

> 사용법: 아래 프롬프트 전체를 복사해서 클로드(claude.ai)에 붙여넣고,
> 이 저장소의 `public/style.css`와 `public/index.html` 두 파일을 함께 첨부하세요.
> 결과물(style.css)을 받으면 유진에게 전달 — 검증 후 적용해드립니다.

---

일본어 가족용 쇼핑 메모 PWA 「わすれもの番長 イッチマン」의 디자인을 리뉴얼하려고 해.
첨부한 `style.css`(현재 스타일)와 `index.html`(마크업)을 기반으로, **완성된 style.css 전체 파일**을 만들어줘.

## 컨셉: 스ー퍼 전대물(スーパー戦隊) 히어로

- 앱 이름은 「わすれもの番長 イッチマン」 — "잊은 물건을 절대 용서하지 않는 번장 히어로"라는 설정.
- 전대물 특유의 느낌: **히어로 레드**를 메인 컬러로, 골드/옐로 포인트, 대담한 사선(斜線) 스트라이프, 엠블럼 느낌의 배지, 굵고 기울어진 타이틀.
- 유치하지 않게: 실사용 앱이므로 "전대물 오프닝의 에너지"를 **포인트 요소에만** 쓰고, 본문/리스트는 읽기 편하게.
- 다크 모드는 "나이트 미션" 느낌(어두운 네이비 + 네온 레드/골드), 라이트 모드는 "출동! 주간 미션" 느낌(밝은 배경 + 강한 레드).

## 화면 구성 (첨부한 index.html 참고)

- 헤더: 로고 + 앱 이름(イッチマン + わすれもの番長 서브타이틀) + 공유📤/설정⚙️ 버튼
- 탭 3개: 買うもの / ストック / 完了 (각 탭에 카운트 뱃지)
- 장소 필터 드롭다운 (`#storeFilterSelect`)
- 리스트: 장소별 그룹(`.store-group`, `.store-head`), 카드(`.list-card`), 행(`.item`)
  - 급한 물건 그룹 `.urgent-head` / `.item.urgent` — 여기는 "경보 발령" 느낌으로 강조
  - 재고 행 `.stock-item`, 수량 컨트롤 `.stock-ctrl`, 재고 부족 `.stock-low` — 부족하면 "위험!" 느낌
- 플로팅 추가 버튼 `#fab` — 히어로의 필살기 버튼처럼 존재감 있게
- 바텀 시트 `dialog.sheet` (추가/편집 폼, 설정), 가족 참가 다이얼로그 `#familyDialog`
- 토스트 `.toast`, 배너 `.banner`

## 반드시 지킬 것 (기술 제약)

1. **선택자(클래스명/ID)는 하나도 바꾸지 말 것** — 기존 style.css에 있는 선택자 구조를 그대로 유지하고 시각 값만 변경. 새 장식용 선택자 추가는 금지 (HTML을 못 바꾸므로 ::before/::after 의사요소로만 장식 추가 가능).
2. **CSS 변수 시스템 유지**: `:root { --teal: ... }` 방식을 유지하되 변수 값을 새 팔레트로 교체. 변수명 자체는 유지해도 되고(값만 히어로 레드로), 새 변수를 추가해도 됨.
3. **라이트/다크 모드 둘 다**: `@media (prefers-color-scheme: dark)` 오버라이드 유지. 두 모드 모두 WCAG AA 대비(본문 4.5:1) 확보.
4. **iOS 필수 사항 유지**: input/select/textarea의 `font-size: 16px` 이상 (자동 확대 방지), `env(safe-area-inset-*)` 패딩, `-webkit-tap-highlight-color: transparent`, `touch-action: manipulation`.
5. 폰트는 시스템 일본어 폰트 스택 유지 (외부 폰트 로드 금지 — 오프라인 PWA임). 타이틀의 임팩트는 font-weight 900 / italic / letter-spacing / text-shadow로 표현.
6. 외부 이미지·외부 리소스 참조 금지. 장식은 CSS gradient / clip-path / 의사요소만 사용.
7. 애니메이션은 가볍게 (transform/opacity만, 0.3초 이하). `prefers-reduced-motion` 대응 추가하면 가점.
8. 터치 타깃 44px 이상 유지.
